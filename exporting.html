Exporting an Annotated Dataset#

This guide should help new prospective students export completed datasets that are going to be used for training the Yolo model.

Generating a New Version of the Dataset#

Well first things first, you have to choose which dataset you want to generate a new version for exporting.

Click the Versions tab in the left-hand toolbar.

Click Generate New Version.

If for some reason you need to, you can add more source images to the dataset in step 1.

Step 2, you can split the dataset into training, validation, or testing.

###Manual Folding Datasets

So, in the past, we would split the dataset into 90% training, 10% validation. We would then just export that into Yolov5 data and use it as is (after some augmentations, explained in just a moment).

###Automatic Folding Datasets

Now, because of cross-validation purposes, there are scripts to fold a dataset into N folds, which will separate sections of the data into the validation set for you. So if you plan/choose to use this method, then you want to keep the dataset 100% training.

We haven’t really done any preprocessing options on Roboflow, so I won’t cover step 3.

Step 4, augmentations.

If you are using Cocosplit, you want to have no augmentations.

If you are using “Robosplit” method, you want to have one augmentation. You want to crop the photos, with a 76%-78% zoom.

Step 5, generating the dataset. Here, if you chose to do “Robosplit”, you want to make 3 images per augmentation.

Exporting a Version#

Once you have created a new version of the dataset, you want to click export to convert it into usable data.

If you are using the Cocosplit script, you will want to export it as COCO data.

If you are doing it the old-fashioned way with Robosplit or anything else, you will want to export it as YOLO v5 Pytorch.

Simply copy the code that is generated, and paste it into a linux terminal to copy over the exported data. The destination of where you want to copy the data over depends on if you are using Cocosplit or if you are using Robosplit.
